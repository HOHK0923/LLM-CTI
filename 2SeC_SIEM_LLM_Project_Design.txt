2SeC SIEM 심화 프로젝트 설계 문서

본 문서는 DVWA 기반 공격 로그를 활용하여
C 언어로 로그를 정제하고,
LLM이 학습 가능한 데이터셋을 구축하여
SIEM + SOAR + LLM 연동까지 이어지는 전체 파이프라인을 설명한다.

--------------------------------------------------
1. 프로젝트 목적
--------------------------------------------------
- 실제 공격 로그 기반 데이터셋 구축
- 공격 타임라인을 이해하는 LLM 설계
- 보안 분석 자동화(SIEM/SOAR) 실험

--------------------------------------------------
2. 전체 아키텍처
--------------------------------------------------
[공격 로그]
  → [C 기반 로그 정제 엔진]
  → [JSON 데이터셋]
  → [RAG 분석]
  → [Fine-tuning]
  → [자동 대응 판단]

--------------------------------------------------
3. C 언어의 역할
--------------------------------------------------
- 텍스트 로그 파싱
- 공격 이벤트 구조화
- 세션 단위 타임라인 생성
- LLM 입력용 JSON 생성

--------------------------------------------------
4. 로그 정제 예시
--------------------------------------------------
입력 로그:
2025-12-12 10:39:15 | SQL_INJECTION | SUCCESS | ' AND SLEEP(3)

출력 JSON:
{
  "timestamp": "2025-12-12T10:39:15",
  "attack_type": "SQL_INJECTION",
  "success": true,
  "payload": "' AND SLEEP(3)"
}

--------------------------------------------------
5. 학습 데이터 구성
--------------------------------------------------
- 입력: 공격 타임라인 JSON
- 출력: 공격 단계, 위험도, 대응 전략

--------------------------------------------------
6. LLM 학습 전략
--------------------------------------------------
1) RAG 기반 분석
2) 로그 누적
3) LoRA 기반 Fine-tuning

--------------------------------------------------
7. 기대 효과
--------------------------------------------------
- SIEM 자동 분석
- 공격 흐름 인식
- 실무형 보안 + AI 포트폴리오

--------------------------------------------------
8. 결론
--------------------------------------------------
본 프로젝트는 공격 로그를 단순 분석이 아닌
'학습 가능한 보안 자산'으로 전환하는 것을 목표로 한다.
